## Reverse Engineering / Binary Exploitation (Pwn)

### Aufgabenstellung

Pwn steht ursprünglich für „own“ bzw. „to own“, also ein System vollständig kontrollieren / kompromittieren. Der Begriff stammt aus der Hackerszene und wurde durch einen Tippfehler von „own“ zu „pwn“.

Eine Pwn-Challenge besteht typischerweise darin:

- ein kompiliertes Programm (Binary) zu analysieren  
- eine Speicher- oder Logikschwachstelle zu finden  
- diese Schwachstelle auszunutzen, um eigenen Code auszuführen oder die Kontrolle über das Programm zu übernehmen

## Übung (Binärdaten)

### Aufgabenstellung

Manche Files, z.B. compilierte Programme, sind Binärdaten. Diese enthalten alle möglichen Bytewerte nicht nur die gültigen ASCII Codes. Eine Anzeige mit cat ist daher nicht zielführend.

Der Inhalt von Binärfiles wird als Hexdump angezeigt. Dies kann mit dem Tool xxd gemacht werden. Ein Hexdump zeigt üblicherweise 3 Spalten an: Offset, Hexcodes der Daten und ASCII Darstellung der Daten.

Probiere:

```bash
$ xxd pwcheck
```

bzw.

```bash
$ xxd pwcheck | more
```

## Übung (GNU Debugger)

### Aufgabenstellung

In diesem Schritt analysieren wir gemeinsam das Programm um einen Weg zu finden die Passwortabfrage zu umgehen.

Zuerst betreiben wir Reverse Engineering, dies bedeutet, eine bestehende Software zu analysieren, um zu verstehen, wie sie funktioniert.

Wenn wir verstanden haben was das Programm tut und wie dessen binäre Struktur aussieht ändern wir das Programm im Speicher bevor es ausgeführt wird.

Wie verwenden für diesen Schritt den GNU Debugger (gdb). Mit einem Debugger kann man einem Programm live bei der Ausführung zusehen und dieses kontrollieren.

---

### GNU Debugger (gdb) – Cheat Sheet

```bash
b main              # set a breakpoint at function main
run                 # start the program from the beginning
disas               # disassemble the current function
disas main          # disassemble the function main
display/i $pc       # show current instruction at every step

x/i 0xabcd          # show the instruction at address 0xabcd
x/5i 0xabcd         # show 5 instructions starting at address 0xabcd
x/16xb 0x401000     # show 16 bytes of data at address 0x401000 in hex

c                # continue program execution
s                # execute the next instruction and step into functions
n                # execute the next line but step over function calls
si               # step into assembler instruction
ni               # step over assembler instruction

info registers eflags # show flags
# set memory
set {unsigned char}0x401000 = 0x90
```

## Übung (jne)

### Aufgabenstellung

Statt auf nop patche den **je (jump if equal)** Befehl auf einen **jne (jump not equal)** Befehl.  
Hintergrund: der **jne** Befehl macht genau das Gegenteil des **je**, damit entsteht das gleiche Ergebnis wie durch das nop-patching.

Teste zuerst im GNU Debugger und fixiere den Patch dann im Programm-Binary.

---

### Test im GNU Debugger

Starte das Programm im Debugger:

```bash
$ gdb ./pwcheck
````

Breakpoint setzen und Programm starten:

```bash 
(gdb) b main
(gdb) run
```

Disassembly anzeigen und Stelle mit `je` suchen:

```bash 
(gdb) disas main
```

---

### Patch im Speicher (je → jne)

Ersetze den Opcode des `je` Befehls durch `jne`:

```bash 
(gdb) set {unsigned char}0x000055e96ccc31c8 = 0x85
```

---

### Programm weiter ausführen

```bash 
(gdb) c
```

```

## Übung (GNU Debugger)

In diesem Schritt analysieren wir gemeinsam das Programm um einen Weg zu finden die Passwortabfrage zu umgehen.

Zuerst betreiben wir Reverse Engineering, dies bedeutet, eine bestehende Software zu analysieren, um zu verstehen, wie sie funktioniert.

````

---

### if-Anweisung finden

Im Disassembly wird eine Stelle mit einem Vergleich und Sprung sichtbar:

```
je =  0x000055625dbf31c8.
```


---

### Manipulation der if-Anweisung (nur Variante 2)

Die einzige verwendete Methode ist das Ersetzen von **je → nop (NOP-Patching)**.

```bash
(gdb) set {unsigned char}0x000055625dbf31c8 = 0x90
(gdb) set {unsigned char}0x000055625dbf31c9 = 0x90
(gdb) set {unsigned char}0x000055625dbf31cA = 0x90
(gdb) set {unsigned char}0x000055625dbf31cB = 0x90
(gdb) set {unsigned char}0x000055625dbf31cC = 0x90
(gdb) set {unsigned char}0x000055625dbf31cD = 0x90
```

---

### Programm weiter ausführen

```bash
(gdb) c
```

---

### Ergebnis

Die if-Abfrage wird durch NOP-Patching entfernt, dadurch wird die Passwortabfrage im Programm umgangen.
